## 세션: v0.9.3 성급한 릴리스의 대가 (역사적 기록)

**날짜:** 2025-12-20

### 배경

v0.9.3 개발 중, 검증을 충분히 하기 전에 릴리스를 재촉하면서 실수가 겹쳤습니다.

### 저지른 실수

#### 1. 확인 없이 가정

**문제:** `__init__.py`의 `__version__`이 `pyproject.toml`과 맞는다고 **가정**함.

**교훈:** 버전 관련 작업 전에는 **두 파일을 모두** 확인(grep)하세요. **반드시 동기화**해야 합니다.

#### 2. 실패를 이해하지 않고 CI만 믿음

**문제:** `.github/workflows/validate.yml`에 mypy·통합 테스트에 `continue-on-error: true`가 있어 실패가 가려짐.

**교훈:** **워크플로 전체를 읽으세요.** `continue-on-error`면 빨간 X가 머지를 막지 않습니다. **침묵하지 말고 고치거나 제거**하세요. (지금은 필수 단계가 실패하면 잡이 실패해야 합니다.)

#### 3. push 전에 로컬 검증 생략

**문제:** CI가 걸러줄 거라 믿고 push했는데, CI는 실패를 무시하도록 설정돼 있었음.

**교훈:** push 전 **항상 로컬에서:**
```bash
uv run ruff check . && uv run ruff format --check . && uv run mypy src/
```

#### 4. 파일을 읽기 전에 수정

**문제:** 내용을 가정하고 수정을 시도함.

**교훈:** 특히 워크플로, `pyproject.toml`, `__init__.py`는 **통째로 먼저 읽기**.

#### 5. 통합 테스트 실패 상태로 릴리스 (역사적)

**문제:** v0.9.3은 **통합 테스트가 실패한 상태**(CI의 무인가 Brave API, PR 포트 오타)로 나감.

**교훈:** **체크가 실패하면 릴리스하지 마세요.** CI를 고치거나 배포를 미룹니다. (이후 프로젝트는 **자동 테스트를 제거**하고 정적 분석·런타임 강제·수동 검증만 씁니다 — `AGENTS.md`의 테스트 제거 로그 참고.)
